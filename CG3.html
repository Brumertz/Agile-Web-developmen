GROUP OF DEFINITIONS 1

Authentication verifies a user's identity, while web security protects data through techniques like Multi-Factor Authentication (MFA), encryption, and SSL/TLS. HTTP is the protocol for transferring web pages. Session management maintains user state across requests using cookies or server-side storage. Stateless programming treats each request independently, enhancing scalability but requiring additional session management.

Authentication is verifying the identity of a user or system. Web security involves protecting data and systems from threats. Techniques include:

  • Multi-Factor Authentication (MFA): Requiring multiple forms of verification.
  • Encryption: Protecting data with cryptographic techniques.
  • Secure Sockets Layer (SSL)/Transport Layer Security (TLS) Encrypting data in transit.

HTTP is the protocol used for transferring web pages on the internet.

It defines how messages are formatted and transmitted and how web servers and browsers should respond to various commands.

Session management is maintaining state information about a user across multiple requests. Techniques include:

  • Cookies: Storing session data on the client side.
  • Session Storage: Using server-side storage to maintain session state.

Stateless programming means each request is independent and does not rely on previous interactions.

This approach improves scalability and reliability but requires additional mechanisms for session management.
